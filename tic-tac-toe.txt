Setup
- Ran "git clone https://github.com/Ethjin8/assign3.git" in terminal, cloning the forked assignment repository onto my local device.


Development
- I followed the React tutorial at this link: https://react.dev/learn/tutorial-tic-tac-toe

- Scaffolding:
  - Installed node.js
  - Used the "npm install" command to install all necessary packages
  - Then, I set up the foundation of my app
  - I first defined the main Board function. Each row was split into separate divs, and each div contained three squares
  - Then, I defined my Square functional component, which returned a button that uses styling outlined for the "square" class in styles.css

- State Management:
  - As the tutorial stated, rather than letting the Square handle its state, I had the Board manage state for all 9 Squares
  - The Board tracked the Squares' contents through an array of 9 values, which were set to null by default
  - It also tracked what turn to take ("X" or "O") through a boolean state value: xIsNext
  - Each Square had two props passed down: its value, as well as a function that was triggered by the onClick event handler
    - The value prop was whatever corresponding value the Board's array had
    - The "handleClick" function prop that was passed down was defined in the Board, with its functionality outlined below:
      - Takes in one parameter: the array index, i
      - If squares[i] already has a value, does nothing
      - Otherwise, a copy of the array of Squares is created
      - Sets the value of the square at that index to either "X" or "O" based on the turn
      - Toggles xIsNext and calls setSquares with the new array, which re-renders the board with the changed values

- Determining Winners:
  - I defined a function, calculateWinner, that took in the squares array and matched it against all possible winning combinations
  - If there was a winning combination, it returned the letter that was there (either "X" or "O")
  - Otherwise, it returned null
  - Earlier, I wanted nothing to happen if squares[i] already had a value. I also wanted nothing to happen if there was a winner, so I added that check at the top of handleClick alongside the existing one
  - Then, I determined the status: if there was a winner, I set the status to "Winner: (X or O)". If there was no winner yet, I set the status to "Next player: (X or O)"
  - I rendered the status above the entire board
